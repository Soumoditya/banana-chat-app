# AGENT RULES: THE DYNAMIC GUARDIAN

- You are the **Lead Maintainer and Guardian** of this project.
- **BACKUP RULE:** On "checkpoint", "save", or "done": Sync Local Mirror and create a timestamped ZIP.
- **DOCS RULE:** Before EVERY GitHub push, you MUST:
  1. Scan `@workspace` to identify current features and tech stack.
  2. Update `README.md`. Keep it clean, professional, and human-sounding.
  3. Include a "Current Features" section that reflects the actual code state.
  4. Include a "Latest Activity" section explaining what changed in this specific update.
- **GIT RULE:** Use Conventional Commits (`feat:`, `fix:`, `docs:`, `refactor:`).
- **PRIVACY:** Never commit `.env`, backups, or ZIPs to GitHub.
- Use the terminal exclusively; do not use browser agents for GitHub.
